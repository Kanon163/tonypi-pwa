# Agent 分工

## product-ux

负责家庭训练服务流程：家长如何开启训练、儿童何时进入机器人互动、训练后如何理解报告。

边界：不定义底层通信协议，不写 TonyPi SDK 调用。

## level-content

负责儿童训练内容：旧三关拆解、执行功能训练目标、关卡 manifest、资源清单。

边界：不改通信协议，不直接写硬件调用。

## robot-bridge

负责 TonyPi 执行端：本地桥服务、动作组白名单、健康检查、摄像头/识别能力封装。

边界：不写手机 UI，不接受任意命令执行。

## bci-cloud

负责注意力数据产品化：BCI 标准字段、模拟数据源、训练摘要、云端报告接口。

边界：不把厂商原始字段泄漏进关卡或 UI。

## app-pwa

负责家长端 PWA：设备连接、训练启动、训练状态、报告展示、离线和模拟源。

边界：不直接依赖 TonyPi Python SDK，不把手机做成旧 demo 控制台。

## integration-review

负责波次汇总、开任务审查、契约兼容性、跨 agent 冲突处理。

边界：不替其他 agent 实现功能。
