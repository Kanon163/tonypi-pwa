# ADR 0001: 手机 PWA 与 TonyPi 桥服务分离

## 状态

采纳。

## 背景

旧 demo 的主程序运行在 TonyPi 上，电脑只适合作为 pygame 控制台。新目标是安卓手机 PWA 产品 demo，并需要接入 BCI、云端报告和可重设计关卡。

## 决策

采用家长端 PWA 作为服务入口，TonyPi 作为儿童训练的主要交互对象。TonyPi 上运行轻量桥服务，PWA 与桥服务通过 `RobotCommand` / `RobotEvent` 通信。

## 后果

- 前端可以独立开发和演示。
- 机器人能力可以用模拟桥替代，便于无硬件开发。
- 需要额外处理网络发现、连接恢复和安全停机。
