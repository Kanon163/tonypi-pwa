# 产品演示闭环 UX 事实审计

## 审计边界

- 本文只基于 `docs/PROJECT_BRIEF.md` 和 `docs/ARCHITECTURE.md`。
- 本波不画高保真 UI、不选框架、不定义底层协议、不实现。
- PWA 定位为家长端服务入口，不是旧 pygame demo 控制台。

## 已确认事实

- 产品由 BCI 头环、TonyPi 机器人、安卓 PWA 组成。
- 家长使用 PWA 连接设备、开启训练、查看摘要和报告。
- 儿童佩戴 BCI 头环，主要与 TonyPi 完成训练互动。
- PWA 接入 BCI 蓝牙数据，记录必要指标。
- BCI 数据可影响 TonyPi 行为，但必须经过干预策略层。
- 训练详细报告由手机上传云端分析，结果回到 PWA。
- 云端分析不参与实时控制闭环。
- 任一设备异常时，首版必须有可解释降级路径。

## 一次训练事实流程

### 训练前

1. 家长打开 PWA，进入训练准备。
2. PWA 引导连接 BCI 头环。
3. PWA 引导连接 TonyPi 桥服务。
4. PWA 展示设备状态、儿童档案或本次训练信息。
5. 家长确认开始训练。

### 训练中

1. 儿童离开手机主视线，与 TonyPi 互动。
2. 关卡引擎推进训练状态，产生 `SessionEvent`。
3. PWA 接收 BCI 样本，记录 `BciSample`。
4. 干预策略层基于 BCI 指标调整提示、节奏、难度或机器人反馈。
5. TonyPi 通过桥服务执行动作、语音或反馈。
6. PWA 只展示家长需要理解的过程摘要和异常状态。

### 训练后

1. PWA 汇总会话事件和 BCI 摘要。
2. PWA 生成 `ReportRequest` 上传云端分析。
3. 云端返回报告结果。
4. PWA 展示家长可理解的训练摘要或报告雏形。

## MVP 屏幕清单

- 训练准备：显示本次训练、设备连接入口、开始条件。
- 设备连接：BCI 头环状态、TonyPi 桥服务状态、重试入口。
- 训练进行：家长端摘要、剩余步骤、设备健康状态、停止训练。
- 异常处理：BCI 异常、TonyPi 不可用、网络异常、报告上传失败。
- 训练完成：本次完成情况、注意力摘要、机器人互动结果、报告状态。
- 报告详情：云端分析结果、家长可读解释、下次训练建议占位。

## 关键 UI 状态

- 未连接：显示需要连接的设备和原因。
- 连接中：显示当前连接目标，允许取消或重试。
- 可开始：BCI、TonyPi、训练内容均满足开始条件。
- 训练中：强调儿童与 TonyPi 互动，PWA 保持家长监看角色。
- 降级中：说明缺失设备对训练和报告的影响。
- 已完成：展示本次训练摘要和报告生成状态。
- 待上传：本地已记录，等待网络或云端恢复。

## 异常和降级事实

- Web Bluetooth 支持存在安卓浏览器差异，BCI 连接失败必须可解释。
- PWA 与 TonyPi 在 AP/LAN 模式下可能发现失败或连接不稳。
- BCI 数据不得直接驱动高风险机器人动作。
- TonyPi 不可用时，PWA 不应伪装为完整儿童训练主舞台。
- 云端报告失败时，应保留本地会话记录并显示待上传状态。

## 演示脚本

1. 家长打开 PWA，看到训练准备页。
2. 服务人员辅助完成 BCI 和 TonyPi 连接。
3. 家长点击开始训练。
4. 儿童佩戴 BCI，与 TonyPi 完成一段训练。
5. PWA 显示训练进行摘要和设备健康状态。
6. 训练结束后，PWA 上传报告请求。
7. PWA 展示本次训练摘要或报告雏形。
8. 演示一个设备异常，并展示可解释降级路径。

## 契约影响

- `SessionEvent` 需要覆盖训练开始、阶段变化、训练结束、异常、降级、上传状态。
- `BciSample` 需要支持家长端必要指标和策略层输入，但 UI 不读取厂商原始字段。
- `ReportRequest` 需要包含会话事件、BCI 摘要、儿童档案标识和应用版本。

## 残余风险

- 任务输入未提供具体儿童训练关卡内容，演示脚本只能定义服务流程。
- 任务输入未提供真实 BCI 厂商能力，连接和指标展示只能定义产品状态。
- 任务输入未提供 TonyPi 桥服务发现方式，连接页只能定义体验需求。
