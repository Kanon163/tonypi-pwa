## 必须修正项

1. 家长端主流程暴露“模拟 / mock / 后续阶段接入”等工程文案。
   - 影响：不符合家长端产品体验，削弱可交付 demo 的可信度。
   - 位置：设备连接、报告服务、报告提示等。
   - 修正：保留演示事实，但改为服务人员可理解的“演示设备 / 演示报告”，或移入调试区。

2. 训练运行页把演示控制直接放在家长主界面。
   - 影响：“TonyPi 切换提示 / 收到扫描成功 / 提前动了”等按钮更像操作台，容易让手机成为训练舞台。
   - 修正：区分“家长监看”与“演示控制”；演示控制可折叠、标注为服务人员模式，默认不抢主视线。

3. 报告摘要数值与训练过程状态存在不一致。
   - 影响：完成页显示 BCI 有效采样率 100%、低信号 0；报告重点仍写 92% 和短暂信号波动，家长会困惑。
   - 修正：报告摘要必须使用本次 session 计算结果，fixture 文案只作 fallback。
